# Summary format

```
内容:

{どんな実装を行ったのか簡潔に書く。before/after を明記し、抽象的な説明が好ましい。}

振る舞い:

{実装の結果、システムがどのように振る舞うべきかを書く。検証手順も明記する。変更がない場合は変更なしで OK。}
```

## 例

```
内容:

ログイン失敗時に平文のエラー詳細 (DB 例外メッセージ) をレスポンスに含めていた問題を修正。
before: catch したエラーをそのまま JSON で返却していた。
after: ユーザー向けには "認証に失敗しました" の固定メッセージのみ返却し、詳細はサーバーログにのみ出力する。

振る舞い:

ログイン失敗時のレスポンスボディに内部エラー情報が含まれなくなる。
検証: 誤ったパスワードでログインし、レスポンス JSON に stack trace や SQL が含まれないことを確認する。
```
