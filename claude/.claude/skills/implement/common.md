# プロジェクトに関係なく守るべき規約

1. コーディング規約

- コメントの内容は WHY / WHY NOT を書く
- WHAT は変数名・シグネチャ・テストからわかるようにする。

2. 実装中コードのコンテキストを意識する

実装レイヤー・抽象化レイヤーを意識する

- 実装レイヤーにおいて過度な抽象化をしてはいけない。シンプルさを優先すべき (YAGNI/KISS)
- 抽象化レイヤーはなるべく宣言的に記述し、副作用を持たない方が好ましい。

- 例: Web アプリケーションの場合
  - 抽象化レイヤー = ドメインやアプリケーション
  - 実装レイヤー = プレゼンテーションやインフラストラクチャ
