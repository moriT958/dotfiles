# Refactor

## 1. Candidates

リファクタリング候補を探し、ユーザーに提案する。

- 重複 => 同じコードが 3 箇所以上で使用されている場合、共通化する。
- 長いメソッド => プライベートメソッドとして分割する。
- Shallow Module => 1 つに組み合わせる、もしくは責務ごとに分割して深くする。
- Feature envy => データに合わせて処理を移す。
- Primitive obsession => 値オブジェクト・Enum・型エイリアスを導入する。

候補がない場合は、以下はスキップ。

## 2. Existing Code

リファクタリングを行った際に、既存コードの問題も見つかった場合はそこもリファクタリング対象とする。

## 3. Explanation

- [ ] そのリファクタリングをしないとどんな問題があるか説明する
- [ ] before/after を明記する
- [ ] インタフェースや振る舞いを変えてないか確認する
